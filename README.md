📁 Projects

📊 Project 1 — Power BI Dashboard: GP Retirement Case Management
Tools: Power BI, Power Query, Excel
Domain: NHS-Linked GP Pensions (Capita)
What I did:

Received raw retirement application data with inconsistent date formats, repeated rows, blank entries and irrelevant columns
Cleaned and transformed the dataset in Power Query — removing duplicates, standardising formats, replacing nulls and renaming columns for clarity
Built a full interactive Power BI dashboard including:

Application volume overview
Processing time analysis
Reasons for application delays
Breakdown by retirement application type


Applied UX principles (clear layout, role-based views, slicers) to make the dashboard accessible to non-technical users

Outcome:
Identified the key delay factors in application processing. The insights supported a workflow redesign that led to a 20% reduction in average processing time.

📋 Project 2 — Agent Productivity Review: Exion Task Performance Analysis
Tools: Excel (XLOOKUP, Pivot Tables, Conditional Formatting), Power Query
Domain: Operations / Performance Management (Capita DSA)
What I did:

Received an operational dataset extracted from the F&A system, listing tasks completed by agents with timestamps
Identified a data privacy risk (employee IDs still present) and mitigated it by anonymising the dataset before any processing
Used XLOOKUP to map each task against benchmark Average Handling Times (AHTs)
Calculated agent productivity by comparing actual task durations against targets
Built pivot tables and pivot charts to visualise performance by agent and by task type
Applied conditional formatting to flag agents performing below, at, or above benchmark

Outcome:
Delivered a clear productivity report to the line manager and training team, enabling data-driven decisions on where coaching and process improvements were needed. Improved overall data accuracy by 25% through validation methods applied during the clean-up.

🏗️ Project 3 — Organisational Data Architecture Mapping
Tools: SSMS (SQL Server Management Studio), Azure Data Factory, Power BI
Domain: Data Architecture & Governance (Capita DSA)

Mapped out Capita's end-to-end data architecture for the DSA account

📈 Project 4 — Statistical Analysis & Forecasting: Call Volume & SLA Modelling
Tools: Excel (Data Analysis Toolpak, Regression, Quartile Functions), Power BI
Domain: Operational Analytics / Capacity Planning (Capita DSA)
What I did:

Cleaned and formatted a daily calls dataset, using quartile functions to detect and handle outliers
Ran descriptive statistics to understand the distribution and spread of call volumes and SLA performance
Built a Multiple Linear Regression model to quantify the relationship between:

Average Handling Time (AHT in seconds) and SLA performance
Total call volume and SLA performance


Assessed multicollinearity using VIF (Variance Inflation Factor) and validated results through the ANOVA table and coefficient analysis
Built a Time-Series Forecasting model, comparing Moving Average (MA3 and MA4) against Exponential Smoothing using error metrics (MAE, RMSE) to select the best-fit model
Visualised the final forecast alongside actuals in a time-series chart

Outcome:
Provided the planning team with a statistically grounded model for predicting future call volumes and their likely impact on SLA — supporting more confident, proactive resource planning decisions.
